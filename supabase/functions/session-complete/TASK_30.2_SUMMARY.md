## Implementation Notes

### Article Generation Flow
1. **Session Completion**: User completes lesson (all milestones or explicit end)
2. **Summary Generation**: session-summarizer creates lesson summary
3. **Session Update**: Status set to "completed", summary stored
4. **Article Generation**: article-generator creates markdown article
5. **Article Storage**: Markdown uploaded to Supabase Storage
6. **Database Record**: Article record inserted into lesson_articles
7. **Session Update**: article_path and article_generated_at added to session

### Graceful Degradation
If article generation fails:
- Session still completes successfully
- Summary is stored
- Response includes warning message
- Article field is null in response

This ensures lesson completion is not blocked by article generation failures.
